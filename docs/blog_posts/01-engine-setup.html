Setting Up a Modern C++ Game Engine

From Zero to Compilation | AIEngine2D Development Blog

This is the first blog post in a series documenting the development of AIEngine2D, a lightweight SNES-inspired 2D game engine built with modern C++17. In this post, we'll cover everything needed to set up the development environment on Windows.

1. Prerequisites

Before we begin, make sure you have the following software installed on your Windows machine:

Software Version Purpose
Visual Studio 2022 17.0+ C++17 compiler (MSVC)
CMake 3.16+ Build system generator
Git 2.x Version control
VS Code (optional) Latest IDE with C++ extensions
Visual Studio Workloads

When installing Visual Studio, make sure to select the "Desktop development with C++" workload. This includes the MSVC compiler and Windows SDK.

Verify CMake Installation

Open a terminal and run:

cmake --version

Expected output: cmake version 3.x.x

2. Installing vcpkg Package Manager

vcpkg is Microsoft's C++ package manager. It simplifies dependency management significantly.

1

Clone vcpkg

cd C:\
git clone https://github.com/microsoft/vcpkg.git
cd vcpkg
2

Bootstrap vcpkg

.\bootstrap-vcpkg.bat

This compiles the vcpkg executable. It may take a few minutes.

3

Integrate with Visual Studio (optional)

.\vcpkg integrate install

This allows Visual Studio projects to automatically find vcpkg packages.

Important Path

This guide assumes vcpkg is installed at C:\vcpkg. If you install it elsewhere, update the paths in the build commands accordingly.

3. Installing Dependencies

AIEngine2D uses three main dependencies:

Package Version Usage
SDL2 2.32.10 Window, input, audio backend
ImGui 1.91.9 Editor UI (with docking + OpenGL3)
OpenGL System Rendering (comes with Windows)

Install with vcpkg

cd C:\vcpkg

# Install SDL2 (64-bit)
.\vcpkg install sdl2:x64-windows

# Install ImGui with SDL2 and OpenGL3 bindings
.\vcpkg install imgui[sdl2-binding,opengl3-binding]:x64-windows
Installation Time

The first installation may take 10-20 minutes as vcpkg downloads and compiles the packages. Subsequent builds are much faster thanks to caching.

Verify Installation

.\vcpkg list

You should see SDL2, ImGui, and their dependencies listed.

4. Cloning the Repository

cd C:\Users\YourName\Documents
git clone https://github.com/SebastienFeser/AIEngine2D.git
cd AIEngine2D

5. Building the Project

1

Create Build Directory

mkdir build
cd build
2

Configure CMake

cmake .. -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake

This generates Visual Studio project files and configures vcpkg integration.

3

Build Debug Configuration

cmake --build . --config Debug
4

Build Release Configuration

cmake --build . --config Release
Build Output

After a successful build, you'll find the executable at:

build/bin/Debug/AIEngine2D.exe (Debug)

build/bin/Release/AIEngine2D.exe (Release)

6. VS Code Configuration

The repository includes pre-configured VS Code settings for an optimal development experience.

Required Extensions

  • C/C++ (ms-vscode.cpptools) - IntelliSense and debugging
  • CMake Tools (ms-vscode.cmake-tools) - CMake integration
  • Shader languages support (slevesque.shader) - GLSL syntax highlighting

Included Configuration Files

File Purpose
.vscode/settings.json Editor settings, CMake configuration, file associations
.vscode/launch.json Debug configurations (Debug + Release)
.vscode/tasks.json Build tasks (Configure, Build Debug, Build Release, Clean)
.vscode/c_cpp_properties.json IntelliSense configuration for C++17

Key Settings

{
    "C_Cpp.default.cppStandard": "c++17",
    "cmake.configureOnOpen": true,
    "cmake.buildDirectory": "${workspaceFolder}/build",
    "cmake.configureArgs": [
        "-DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake"
    ],
    "editor.formatOnSave": true
}

Build Tasks

Use Ctrl+Shift+B to open the build menu:

  • Configure CMake - Generate build files
  • Build Debug - Compile debug version (default)
  • Build Release - Compile optimized version
  • Clean - Remove build artifacts
  • Rebuild Debug - Clean + Build

Debugging

Press F5 to start debugging. Two configurations are available:

  • Debug AIEngine2D - Run with debugger attached
  • Release AIEngine2D - Run release build

7. First Run

After building, run the executable:

.\build\bin\Debug\AIEngine2D.exe

The engine window displays a cornflower blue background (classic DirectX test color)

Expected Console Output

Engine initialized successfully!
Window: 800x600
OpenGL context created
Press ESC to exit

Controls

  • ESC - Exit the application
  • Close button - Exit the application
Phase 0 Complete!

If you see the blue window and console output, congratulations! The engine is set up correctly and ready for development.

Code Style

The project includes a .clang-format file based on Google style with modifications:

  • 4-space indentation
  • Allman brace style (braces on new lines)
  • 100 character line limit
  • Automatic include sorting

Enable "Format on Save" in VS Code to automatically format your code.
